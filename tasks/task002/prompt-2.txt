你实现的分层缓存功能模块存在一些问题，需要修复：
1. get_stats 方法在计算整体命中率时直接将本地缓存和共享缓存的访问次数相加，一次 get 调用若本地未命中而共享命中会被计入两次访问，导致整体访问量虚高而命中率偏低。需要在 LayeredCache 内维护独立的请求级统计计数器。
2. get 方法在共享缓存命中后回填本地缓存时，回填的 TTL 完全忽略共享缓存条目自身的过期时间，直接使用本地默认 TTL 或调用方传入的 local_ttl，可能导致本地副本与共享缓存中的原始 TTL 不一致。回填时应读取共享条目的剩余存活时间。
3. get_or_load 方法返回类型标注为泛型 T 但实际可能返回 None（当 loader 返回 None 时），代码用 `# type: ignore` 注释回避而非正确处理，增加了运行时空值风险。应改为 Optional[T] 返回类型或在 loader 返回 None 时抛出明确异常。
4. get_with_level 方法不支持传入 loader 函数和 TTL 参数，与核心 get 方法的接口存在明显断层，用户无法通过该方法享受与 get 同等的读穿透能力。应使该方法支持 loader 和 TTL 参数。
5. 修复后需要为该模块补充共享缓存 TTL 回填一致性和 get_or_load 返回 None 场景的单元测试。
6. 更新 README.md 说明修复后的统计计算方式、TTL 回填策略和类型安全约定。
7. 重新运行 poetry run pytest tests/layered_cache/ -q，需要让所有测试通过。
